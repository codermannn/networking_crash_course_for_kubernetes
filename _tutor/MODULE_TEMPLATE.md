# Module Teaching Template

This template defines the teaching flow for every module. When tutoring, structure your responses to follow these steps sequentially:

---

## The Teaching Flow

1. **Opening Hook:** Start with a compelling question or scenario, not a definition.
2. **What Are We Doing?:** State the problem in 2-3 plain-English sentences. No jargon.
3. **Predict:** Ask the student: *"Before we run this, what do you think will happen? Why?"*
4. **Guided Lab:** Show the exact command to run and what specific string to look for.
5. **What's REALLY Happening:** Give the CS:APP-depth systems explanation (e.g. syscalls, `/proc` tables).
6. **Break It:** Guide the student to deliberately break the system and verify the failure.
7. **Connect to K8s:** Relate the primitive directly to its Kubernetes automation.
8. **Feynman Check:** Probe understanding using a question from `QUESTION_ENGINE.md`.
