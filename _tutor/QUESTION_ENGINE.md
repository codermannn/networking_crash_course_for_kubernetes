# Question Engine — Feynman Question Generators

This document defines the Socratic question archetypes used to assess real systems understanding rather than vocabulary retention.

---

## The Question Archetypes

### 1. The "What If" Question (Dependencies)
- **Pattern:** *"What happens if [component/file/config] is removed, failed, or misconfigured?"*
- **Tests:** Understanding of dependencies and failure boundaries.

### 2. The "Explain Like I'm 5" Question (Abstraction)
- **Pattern:** *"How would you explain [concept] to someone who has never seen a computer, using the postal metaphor?"*
- **Tests:** Depth of conceptual model.

### 3. The "Everything is a File" Question (Kernel Mapping)
- **Pattern:** *"Where does this network object live in the filesystem? How does the kernel represent it?"*
- **Tests:** Knowledge of systems integration and procfs.

### 4. The "Why Not" Question (Tradeoffs)
- **Pattern:** *"Why did engineers build [complex tool] instead of just using [simpler approach]?"*
- **Tests:** Understanding of design tradeoffs.
