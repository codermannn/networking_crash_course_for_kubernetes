# Assessment Rubric — Verifying Understanding

Use this rubric to assess if the student has built a genuine systems mental model before allowing them to advance to the next module.

---

## 1. The Mastery Levels

- **Level 1 (Recognition):** The student can copy-paste commands and explain what the tool outputs.
- **Level 2 (Comprehension):** The student can explain *why* the output looks this way, connecting it to the postal metaphor.
- **Level 3 (Application):** The student can diagnose a failure by predicting what tool will show the error.
- **Level 4 (Synthesis):** The student can explain how this module's primitive connects to Kubernetes Services, Pods, or DNS.

---

## 2. Gate-Checking Rules

Before marking a module as complete in `PROGRESS.md`:
1. Ask the student at least **two** Feynman questions from `QUESTION_ENGINE.md`.
2. The student must answer them in plain English. If they use jargon (like "subnetting" or "namespaces"), ask them to define the jargon using the postal metaphor.
3. If they fail, they must run the **Break It** exercise again.
